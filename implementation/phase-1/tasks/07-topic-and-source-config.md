# 07. Topic & Source Config

## Цель

Заложить конфигурацию темы MVP и источников так, чтобы новая тема в будущем добавлялась через config, а не переписывание ядра.

## Контекст

Первая тема: мобилизационные и военно-экономические риски для бизнеса. Она должна влиять на набор источников, фильтрацию, тематические связи, скоринг и аналитические сводки.

Тематическая конфигурация также задает правила отбора кандидатов для связей событие-событие. Простое временное следование событий не считается самостоятельной связью: даты, общий актор, ведомство или регион используются как признаки скоринга и ограничения.

## Работы

- Создать YAML-конфиг темы.
- Загрузить конфигурацию через `@ConfigurationProperties`.
- Описать source codes P0.
- Описать keyword list первой темы.
- Описать event types первой темы.
- Описать базовые weights для релевантности.
- Описать базовые relation gates для тематических связей:
  - минимальная тематическая релевантность;
  - допустимое временное окно;
  - признаки вроде общего ведомства, региона, похожей формулировки или общего документа.
- Добавить endpoint чтения активных тем и источников.
- Добавить validation конфигурации при старте приложения.

## Минимальная структура

```yaml
logrus:
  topics:
    - code: mobilization_business_risk
      title: "Мобилизационные и военно-экономические риски"
      enabled: true
      sourceCodes:
        - publication_pravo
        - regulation_gov
        - duma_api
        - government
        - kremlin
        - mintrud
        - trudvsem
      keywords:
        - мобилизация
        - воинский учет
        - бронирование
        - отсрочка
        - оборонный заказ
      relationGates:
        maxTimeWindowDays: 45
        minTopicScore: 0.65
        signals:
          - shared_agency
          - shared_region
          - similar_phrase
          - shared_document
```

## Зависимости

## Результат

- Приложение знает активную тему MVP.
- Источники и ключевые слова не захардкожены в коде обработки.

## Acceptance Criteria

- Конфигурация темы загружается при старте.
- Некорректная конфигурация ломает старт понятной ошибкой.
- API может вернуть список активных тем.
- API может вернуть список источников, включенных в тему.
- В конфигурации есть базовые правила отбора кандидатов связей.
- Конфигурация не описывает `PRECEDES` как самостоятельный тип связи.

## Не входит

- UI-админка тем.
- Хранение конфигурации тем в БД.
- Автоматическая генерация словарей.
