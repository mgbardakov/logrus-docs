# 06. Angular UI Skeleton

## Цель

Создать минимальный Angular frontend для первого рабочего экрана MVP внутри репозитория `logrus-bff`.

## Контекст

Первый экран MVP — это не одна “сводка”, а панель аналитика. Она должна быстро отвечать на вопрос: что требует внимания прямо сейчас.

В домене LOGRUS аналитическая сводка — отдельный рабочий артефакт. Сводок может быть много: система готовит их по периодам/темам/накопленным событиям, а аналитик затем проверяет, отправляет на доработку или фиксирует. Поэтому домашняя страница должна показывать рабочий срез по сводкам, а не только последнюю сводку.

Angular приложение живет в `logrus-bff/logrus-ui`. В dev-режиме оно может запускаться отдельным Angular dev server и ходить в `logrus-bff` через proxy. Для packaged MVP результат сборки копируется в `logrus-bff/src/main/resources/static`, чтобы BFF сам раздавал frontend.

## Работы

- Инициализировать Angular приложение в `logrus-bff/logrus-ui`.
- Настроить routing.
- Настроить HTTP client для BFF.
- Создать layout приложения.
- Создать страницу панели аналитика.
- На панели аналитика показать:
  - количество сводок на проверке;
  - количество новых/важных событий;
  - размер очереди разбора;
  - короткий список сводок на проверке.
- Создать страницу списка аналитических сводок.
- Создать компонент списка событий.
- Создать компонент карточки события.
- Создать компонент отображения evidence/первоисточников.
- Создать страницу/заглушку очереди разбора.
- Добавить базовые loading/error states.
- Подготовить dev proxy к BFF.
- Подготовить build output для копирования в `src/main/resources/static`.

## Основные маршруты

```text
/
/briefs
/events
/events/:id
/review
```

## Термины UI

- “Панель аналитика” — домашний overview-экран.
- “Обзор” — пункт навигации на домашнюю страницу.
- “Аналитическая сводка” — доменный артефакт, который может иметь статус и требовать проверки.
- “Сводки на проверке” — список сводок, ожидающих действия аналитика.

## Результат

- UI открывается локально.
- UI получает stub данные из BFF.
- Панель аналитика показывает счетчики и список сводок на проверке.
- Отдельный экран `/briefs` показывает список аналитических сводок.
- Карточка события открывается.
- Собранный UI может быть отдан через `logrus-bff`.

## Acceptance Criteria

- Angular приложение запускается локально.
- Angular приложение находится в `logrus-bff/logrus-ui`.
- Панель аналитика показывает счетчики “Сводки на проверке”, “Новые события”, “Очередь разбора”.
- Панель аналитика показывает короткий список сводок на проверке.
- Экран `/briefs` открывается и показывает список сводок.
- Карточка события открывается.
- Ошибки загрузки отображаются пользователю.
- Пользовательские тексты UI ориентированы на русскоязычную аудиторию.
- Тексты не используют термин “инвестиционные сигналы”.
- Есть понятная команда или инструкция для сборки UI в static resources BFF.

## Не входит

- Финальный дизайн.
- Сложные графы.
- Алерты.
- Полный workflow проверки и редактирования сводок.
- Полный review workflow.
