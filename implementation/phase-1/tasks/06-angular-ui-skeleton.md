# 06. Angular UI Skeleton

## Цель

Создать минимальный Angular frontend для первого рабочего экрана MVP.

## Контекст

Интерфейс MVP должен быть проще BI-панели: сводка, список событий, карточка выбранного события, основания и первоисточники.

## Работы

- Инициализировать Angular приложение.
- Настроить routing.
- Настроить HTTP client для BFF.
- Создать layout приложения.
- Создать страницу dashboard.
- Создать компонент списка событий.
- Создать компонент карточки события.
- Создать компонент отображения evidence/первоисточников.
- Добавить базовые loading/error states.
- Подготовить dev proxy к BFF.

## Основные маршруты

```text
/
/events
/events/:id
/review
```

## Зависимости

- `01-project-scaffold.md`
- `05-logrus-bff-skeleton.md`

## Результат

- UI открывается локально.
- UI получает stub данные из BFF.
- Визуальная структура соответствует упрощенному SVG-макету.

## Acceptance Criteria

- Angular приложение запускается локально.
- Dashboard показывает stub-сводку.
- Карточка события открывается.
- Ошибки загрузки отображаются пользователю.
- Тексты не используют термин “инвестиционные сигналы”.

## Не входит

- Финальный дизайн.
- Сложные графы.
- Алерты.
- Полный review workflow.
