# ЛОГРУС: источники сырых данных для MVP

Дата фиксации: 2026-06-17  
Фокус MVP: мобилизационные и военно-экономические риски для бизнеса.

Базовая единица системы: событие.

## Первый контур

Источники, которые стоит подключать первыми: они дают правовые изменения, проекты НПА, законопроекты, официальную речь и трудовой контекст.

| Источник | Машинный доступ | Что забирать | Приоритет |
|---|---|---|---|
| [publication.pravo.gov.ru](https://publication.pravo.gov.ru/) | API + открытые данные | Официально опубликованные НПА: указы, законы, постановления, приказы, региональные акты | P0 |
| [regulation.gov.ru](https://regulation.gov.ru/projects) | XML API + RSS | Проекты НПА, стадии обсуждения, тексты проектов, карточки документов | P0 |
| [api.duma.gov.ru](https://api.duma.gov.ru/) / [sozd.duma.gov.ru](https://sozd.duma.gov.ru/search) | Официальный API, нужен ключ | Законопроекты, стадии рассмотрения, субъекты внесения, тексты и карточки | P0 |
| [kremlin.ru](https://kremlin.ru/) | RSS + HTML | Указы, поручения, выступления, стенограммы, Совбез | P0 |
| [government.ru](https://government.ru/) | RSS + HTML | Решения правительства, документы, поручения, новости, меры устойчивости экономики | P0 |
| [mintrud.gov.ru](https://mintrud.gov.ru/) | Открытые данные + RSS | Новости, документы, разъяснения по работодателям, труду, воинскому учету | P0 |
| [trudvsem.ru](https://trudvsem.ru/) | Официальный API + открытые данные | Вакансии, рынок труда, деперсонализированные наборы, кадровый контекст | P0 |

## Второй контур

Источники важные, но их можно подключать после первого рабочего пайплайна.

| Источник | Машинный доступ | Что забирать | Приоритет |
|---|---|---|---|
| [budget.gov.ru](https://budget.gov.ru/) | OpenData + API/JSON | Бюджетные классификаторы, расходы, программы, открытые бюджетные наборы | P1 |
| [proverki.gov.ru](https://proverki.gov.ru/) | Открытые данные; закрытая интеграция через ЕСИА/СМЭВ | Проверки бизнеса, контрольные и надзорные мероприятия, статистика проверок | P1 |
| [minpromtorg.gov.ru](https://minpromtorg.gov.ru/) | Открытые данные + HTML | Новости, документы, промышленная политика, ОПК-контекст, реестры промышленной продукции | P1 |
| [mil.ru](https://mil.ru/) | Публичного API не найдено; HTML + Telegram | Новости Минобороны, публичные заявления, сообщения о контрактной службе | P1 |

## Отдельный технический spike

| Источник | Текущий статус | Что проверить |
|---|---|---|
| [zakupki.gov.ru](https://zakupki.gov.ru/epz/main/public/home.html) | Публичный поиск есть, простой актуальный REST API не очевиден | Актуальность XML-выгрузок, наличие новых машиночитаемых интерфейсов, допустимость парсинга поиска, лимиты, структура карточек закупок |

Старый `ftp.zakupki.gov.ru` ранее использовался для XML-выгрузок ЕИС, но на момент проверки не резолвился из локальной среды. Закупки важны для военно-экономического контура, но подключение стоит вынести в отдельное исследование.

## Конкретные точки доступа

### publication.pravo.gov.ru

- Главная: <https://publication.pravo.gov.ru/>
- API-интерфейс: <https://publication.pravo.gov.ru/help>
- Открытые данные: <https://publication.pravo.gov.ru/OpenData>

Использование в MVP: официальный источник принятых правовых актов. Нужен регулярный сбор новых публикаций и фильтрация по темам: мобилизация, воинский учет, оборона, трудовые обязанности работодателей, экономика, промышленность, госзаказ, контроль.

### regulation.gov.ru

- Проекты: <https://regulation.gov.ru/projects>
- Паспорт открытых данных: <https://regulation.gov.ru/opendata/7710349494-npalist/>
- API из паспорта: `http://regulation.pba.su/api/npalist`

Параметры API из паспорта:

- `id` - получить один проект по идентификатору;
- `limit` - размер страницы;
- `offset` - смещение;
- `fields` - список полей;
- `search` - полнотекстовый поиск;
- `sort=desc` - обратный порядок.

Использование в MVP: раннее обнаружение будущих изменений правил.

### api.duma.gov.ru / СОЗД

- API Госдумы: <https://api.duma.gov.ru/>
- Документация: <https://api.duma.gov.ru/pages/dokumentatsiya>
- Обращение к API: <https://api.duma.gov.ru/pages/dokumentatsiya/obrashchenie-k-api>
- Запрос ключа: <https://api.duma.gov.ru/key-request>
- СОЗД: <https://sozd.duma.gov.ru/search>

Использование в MVP: отслеживать законопроекты и стадии рассмотрения. На этапе реализации нужно проверить актуальную работоспособность методов API, потому что на сайте встречались уведомления о технических работах.

### kremlin.ru

- Главная: <https://kremlin.ru/>
- События: <https://www.kremlin.ru/events/president/news>
- Стенограммы: <https://www.kremlin.ru/events/president/transcripts>
- Подписки/RSS: <https://kremlin.ru/subscribe>
- Совет безопасности: <https://special.kremlin.ru/events/security-council>

Использование в MVP: президентские указы, поручения, стенограммы, заседания Совбеза, публичная риторика.

### government.ru

- Главная: <https://government.ru/>
- Подписка/RSS: <https://services.government.ru/subscribe/>
- Раздел по экономике в условиях частичной мобилизации: <https://government.ru/rugovclassifier/903/>
- Меры устойчивости экономики: <https://government.ru/mob_measures/>

Использование в MVP: решения правительства, документы, поручения, меры поддержки/ограничения, управленческий контекст.

### mintrud.gov.ru

- Главная: <https://mintrud.gov.ru/>
- Открытые данные: <https://mintrud.gov.ru/opendata>
- Новости: <https://mintrud.gov.ru/news/news/list>
- Документы: <https://mintrud.gov.ru/docs>
- RSS документов: `https://mintrud.gov.ru/document/rss`

Использование в MVP: кадровые и трудовые обязанности бизнеса, воинский учет работодателей, разъяснения, документы.

### trudvsem.ru

- Главная: <https://trudvsem.ru/>
- API: <https://trudvsem.ru/opendata/api>
- Открытые данные: <https://trudvsem.ru/opendata>
- Наборы данных: <https://trudvsem.ru/opendata/datasets>

Использование в MVP: кадровый контекст, вакансии, изменения спроса на труд, региональная динамика.

### budget.gov.ru

- Главная: <https://budget.gov.ru/>
- Открытые данные: <https://www.budget.gov.ru/Открытые-данные>

В открытых данных встречаются API-эндпоинты вида:

`http://budget.gov.ru/epbs/registry/<dataset>/data?pageSize=100`

Использование в MVP: бюджетные движения, классификаторы, косвенные признаки приоритизации расходов.

### proverki.gov.ru

- Главная: <https://proverki.gov.ru/>
- Публичный поиск: <https://proverki.gov.ru/portal/public-search>
- Открытые данные: <https://proverki.gov.ru/portal/public-open-data>

Использование в MVP: контрольные и надзорные мероприятия, проверки бизнеса, региональные практики.

Важно: полная интеграция ФГИС ЕРКНМ описана через ЕСИА/СМЭВ и не является обычным публичным API для внешнего сервиса. Для MVP использовать только публичные открытые данные.

### minpromtorg.gov.ru

- Главная: <https://minpromtorg.gov.ru/>
- Открытые данные: <https://minpromtorg.gov.ru/opendata>
- Документы: <https://minpromtorg.gov.ru/docs/>
- ГИСП: <https://gisp.gov.ru/>
- Реестр российской промышленной продукции: <https://gisp.gov.ru/pp719v2/pub/prod/>

Использование в MVP: промышленная политика, производственная мобилизация, меры поддержки, реестры, ОПК-контекст.

### mil.ru

- Главная: <https://mil.ru/>
- Официальный Telegram: <https://t.me/s/mod_russia>
- Контрактная служба через Госуслуги: <https://contract.gosuslugi.ru/>
- Отбор на контракт в Минобороны: <https://guk.mil.ru/kontrakt>

Использование в MVP: только публичные новости, заявления и справочные изменения. Не собирать и не обрабатывать персональные данные.

### zakupki.gov.ru

- Главная: <https://zakupki.gov.ru/epz/main/public/home.html>
- Расширенный поиск закупок: <https://zakupki.gov.ru/epz/order/extendedsearch/search.html>
- Результаты расширенного поиска: <https://zakupki.gov.ru/epz/order/extendedsearch/results.html>

Использование в MVP: пока после spike. Потенциально важный источник для оборонно-промышленного и производственного контура, но требует отдельной проверки технического доступа.

## Рекомендуемый порядок реализации

1. Подключить `publication.pravo.gov.ru`.
2. Подключить `regulation.gov.ru`.
3. Подключить `api.duma.gov.ru`.
4. Подключить RSS/HTML для `government.ru` и `kremlin.ru`.
5. Подключить `mintrud.gov.ru` и `trudvsem.ru`.
6. Добавить `budget.gov.ru`.
7. Добавить `proverki.gov.ru`.
8. Провести отдельный spike по `zakupki.gov.ru`.

## Ключевые типы событий

- опубликован правовой акт;
- появился проект НПА;
- внесен или изменен законопроект;
- изменился статус законопроекта;
- опубликовано поручение или решение правительства;
- опубликована стенограмма или важное выступление;
- опубликовано ведомственное разъяснение;
- изменились открытые данные по рынку труда;
- появилась бюджетная или закупочная активность;
- появились проверки или контрольные мероприятия по релевантной теме.

## Тематические фильтры для MVP

Первичные ключевые темы:

- мобилизация;
- мобилизационная подготовка;
- воинский учет;
- бронирование граждан;
- отсрочка;
- повестка;
- реестр повесток;
- контрактная служба;
- оборонный заказ;
- государственный оборонный заказ;
- ОПК;
- промышленная мобилизация;
- гражданская оборона;
- трудовые обязанности работодателей;
- проверки работодателей;
- критическая инфраструктура;
- устойчивость экономики;
- импортозамещение;
- спецрежим;
- чрезвычайная ситуация.

## Принципы безопасности и позиционирования

- Не позиционировать продукт как инструмент обхода законодательства или уклонения от обязанностей.
- Не собирать персональные данные из чувствительных реестров.
- Разделять факт, связь и аналитический вывод.
- Всегда хранить ссылку на первоисточник.
- У каждого события фиксировать источник, дату, тип документа/материала, акторов и уровень уверенности разметки.
- Для выводов использовать формулу: событие -> связь -> основание -> вывод.
